# Contributing to Synkora ERP

Thank you for contributing!

## Getting Started

1. Fork the repository.

2. Clone your fork.

3. Create a new branch from `develop`.

Example

```bash
git checkout develop

git checkout -b feature/member-registration
```

## Branch Naming

Features

feature/member-registration

feature/paystack

feature/dashboard

Bugs

bugfix/login

bugfix/payments

Documentation

docs/readme

Plugins

plugin/mpesa

plugin/ocr

Themes

theme/dark

theme/minimal

## Commit Messages

feat: Added Paystack Plugin

fix: Fixed Member Registration

docs: Updated README

refactor: Improved API Manager

## Pull Requests

- Keep PRs focused.
- Reference related Issues.
- Add screenshots for UI changes.
- Ensure tests pass.

## Code Style

Follow PEP 8.

Use Black for formatting.

Use Ruff for linting.

## Thank You

Every contribution helps Synkora become better.
